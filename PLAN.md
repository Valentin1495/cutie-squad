# 응원단 목표 설정/수정 UX 구현 계획

## Summary
- 첫 진입 온보딩을 “응원단 작전회의” 콘셉트로 구성한다.
- 사용자는 추천 목표 카드 더미와 직접 입력을 통해 오늘 응원할 목표 3-5개를 만든다.
- 목표가 추가될 때마다 작은 캐릭터가 하나씩 합류해 앱의 세계관에 자연스럽게 들어오게 한다.
- 메인 화면에서는 목표 수정 진입점을 제공하고, 완료 상태와 수정 상태가 자연스럽게 이어지도록 한다.

## 핵심 UX 방향
- 가장 중요한 방향은 “추천 목표 카드 더미 + 직접 입력 + 응원단이 하나씩 합류하는 온보딩”이다.
- 귀엽지만 기능을 가리지 않아야 한다.
- 목표를 만드는 순간부터 사용자가 앱 안의 응원단과 함께 작전을 세우는 느낌을 줘야 한다.

## 온보딩: 응원단 작전회의
- 첫 화면에는 작은 캐릭터들이 빈 무대에 모여 있는 장면을 보여준다.
- 말풍선 형태로 “오늘 우리가 뭘 응원할까?”를 노출한다.
- 목표가 하나 추가될 때마다 캐릭터 하나가 톡 튀어나와 해당 목표 카드 옆에 붙는다.
- 목표 개수가 늘어날수록 “응원단이 합류하고 있다”는 피드백이 보여야 한다.

## 목표 입력: 카드 뽑기
- 빈 목표 입력칸을 바로 노출하지 않는다.
- “목표 카드 만들기” 버튼을 먼저 보여주고, 버튼을 누르면 카드가 뒤집히며 입력 폼이 나타난다.
- 직접 입력 카드에는 목표 텍스트 입력과 마스코트/이모지 선택이 들어간다.
- 추천 목표는 카드 더미처럼 보여준다.
- 추천 목표 카드를 탭하면 내 목표 영역으로 “쏙” 이동하는 애니메이션을 준다.
- 기본 추천 목표 예시는 다음을 사용한다:
  - 물 마시기
  - 산책하기
  - 책 읽기
  - 스트레칭
  - 일찍 자기

## 이모지 선택: 마스코트 뽑기
- 목표마다 이모지를 선택하는 UI를 작은 룰렛 또는 슬롯머신처럼 만든다.
- 실제 랜덤 기능이 과할 필요는 없다.
- 선택 시 `💧`, `🏃`, `📚`, `🌙`, `✍️` 등이 좌우로 흔들리며 바뀌는 정도의 가벼운 재미를 준다.
- 이모지 선택은 목표 입력을 방해하지 않아야 한다.

## 최소 목표 개수와 출정 버튼
- 목표는 최소 3개, 최대 5개까지 설정할 수 있다.
- 목표가 3개 미만이면 하단 CTA는 비활성 회색 상태다.
- 목표가 3개 이상이면 CTA가 보라색으로 활성화된다.
- CTA 문구는 “응원 시작하기”를 기본으로 한다.
- CTA가 활성화되는 순간 캐릭터들이 살짝 점프하는 애니메이션을 준다.

## 첫 저장 애니메이션
- CTA를 누르면 목표 카드들이 한 줄로 정렬된다.
- 캐릭터들이 “오늘의 응원판”을 들고 메인 화면으로 전환되는 느낌을 준다.
- 구현은 `framer-motion`의 `layoutId`, `layout`, `AnimatePresence`를 우선 활용한다.
- 애니메이션은 전환감을 주되, 저장 완료와 화면 이동을 늦게 느끼게 만들지 않는다.

## 상태 모델
- `Goal`은 안정적인 `id`, `text`, `emoji`, `done`, `createdAt`을 가진다.
- 저장 상태에는 `hasOnboarded`를 포함한다.
- store 액션은 다음을 제공한다:
  - `completeOnboarding(goals)`
  - `saveGoals(goals)`
  - `updateGoals(goals)`
- 날짜가 바뀌면 기존 목표 텍스트와 이모지는 유지하고 `done: false`만 리셋한다.
- 기존 저장 데이터에 `hasOnboarded`가 없더라도 목표가 있으면 온보딩 완료 상태로 간주하는 마이그레이션을 고려한다.

## 화면 흐름
- `App`은 로딩 완료 후 `hasOnboarded`가 false면 `OnboardingPage`를 렌더링한다.
- `hasOnboarded`가 true면 기존 `MainPage` 또는 `CelebrationPage`를 렌더링한다.
- `CelebrationPage`에서 돌아갈 때는 `resetGoals()`를 호출하지 않고 `showCelebration`만 닫는다.
- 모든 목표 완료 후 메인 화면으로 돌아오면 같은 목표를 다시 체크하지 못하고, 완료 상태 안내를 보여준다.

## 목표 수정 UX
- 메인 화면 상단 또는 목표 목록 근처에 “목표 수정하기” 진입점을 둔다.
- 수정 화면은 온보딩과 같은 편집 경험을 재사용하거나 `GoalEditor` 컴포넌트로 분리한다.
- 기존 목표의 `id`가 유지되는 항목은 `done` 상태를 유지한다.
- 새로 추가한 목표만 `done: false`로 저장한다.
- 삭제된 목표는 저장 목록에서 제거한다.

## Test Plan
- 첫 실행: 저장 데이터가 없으면 “응원단 작전회의” 온보딩이 뜬다.
- 목표 개수: 3개 미만이면 CTA가 비활성화되고, 3-5개면 “응원 시작하기”가 활성화된다.
- 추천 카드: 추천 목표를 탭하면 내 목표 카드로 이동하고 캐릭터가 합류한다.
- 직접 입력: “목표 카드 만들기”를 누르면 카드가 뒤집히며 입력 폼이 나타난다.
- 이모지 선택: 이모지를 바꾸면 슬롯/룰렛 느낌의 작은 흔들림이 보인다.
- 첫 저장: CTA를 누르면 저장 애니메이션 후 메인 화면으로 이동한다.
- 완료 루프: 모든 목표 완료 후 축하 화면에서 돌아와도 같은 목표가 다시 체크 가능 상태로 리셋되지 않는다.
- 날짜 변경: `lastResetDate`가 어제인 저장 데이터를 불러오면 같은 목표가 오늘 미완료 상태로 리셋된다.
- 수정: 완료한 기존 목표는 완료 상태가 유지되고, 새 목표만 미완료로 보인다.
- 검증 명령:
  - `npx.cmd tsc -b`
  - `npm.cmd run build`
